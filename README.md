# TreeChat - 树形结构的大模型对话工具

[![GitHub license](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![C#](https://img.shields.io/badge/language-C%23-green.svg)](https://docs.microsoft.com/en-us/dotnet/csharp/)
[![WPF](https://img.shields.io/badge/UI-WPF-purple.svg)](https://docs.microsoft.com/en-us/dotnet/desktop/wpf/)

TreeChat 是一款采用**树形结构**组织大模型对话的桌面应用，解决传统线性对话中上下文混乱、信息难以追溯的问题。每个分支对话独立展开、互不干扰，让你清晰管理多线程讨论。

## 📌 项目背景

在使用 DeepSeek、豆包等常见大模型应用时，常遇到两个痛点：

- 单次回复包含多个疑问，多次追问后线性对话结构混乱，需要反复上下翻找信息；
- AI 会读取完整上下文，已解决的旧问题持续干扰新回答。

**树形对话结构**天然解决这些问题 —— 每个分支独立存储上下文，切换分支即切换 Prompt 范围。

## ✨ 核心特性

- 🌲 **树形对话结构** —— 完整保留对话分支关系，支持无限嵌套
- 🔍 **精准上下文筛选** —— AI 仅读取当前节点到根节点的路径，旧讨论不干扰新分支
- 🖱️ **可视化树形图** —— 节点连线清晰展示对话脉络，支持拖拽/缩放画布
- 📝 **多对话管理** —— 同时维护多个独立树形对话，侧边栏快速切换
- ⚡ **异步 AI 调用** —— 非阻塞发送消息，界面流畅不卡死
- 🎨 **MVVM 架构** —— 数据与界面解耦，便于维护和扩展

## 🚀 快速开始

### 环境要求

- Windows 7+ / 支持 WPF 的操作系统
- [.NET 6.0 SDK](https://dotnet.microsoft.com/en-us/download) 或更高版本
- 可访问的大模型 API（当前版本只提供华东师范大学内部模型，需本校密钥，后续更新会支持更多模型）

### 安装与运行

1. 确保已符合所需环境
2. release处下载压缩包，解压后.exe即为应用程序
3. 首次使用需要点击更改配置，填入自身的学校模型密钥（前往https://chat.ecnu.edu.cn/html/#welcome 网站申请）
4. 配置完成后，即可开始使用：

### 使用说明

#### 创建新对话

- 点击左侧面板顶部的 **“新聊天”** 按钮，即可创建一个空白对话树（默认名为“新对话”）。
- 左侧列表会显示所有已创建的对话树，单击任意对话可切换当前编辑的对话。
- 退出应用程序时会**丢弃所有对话**，持久化保存需要**手动保存和读取**

#### 发起第一次提问

- 选中一个对话后，在右侧下方的输入框中输入你的问题，点击 **“发送”** 按钮。
- 系统会将该问题作为根节点，并调用 AI 模型返回回答，显示在右侧的消息展示区。

#### 分支对话（核心功能）

- 当 AI 返回的回答中包含多个可追问的方向时，你可以对 **任意一条已有的用户消息节点** 进行分支追问。
- **方法**：在中间树形可视化面板中，单击你想要追问的那个用户消息节点（节点会高亮显示），然后在右侧输入新问题并发送。
- 此时，新问题会成为该节点的 **子节点**，形成独立的分支。AI 在回答这个分支问题时，**只会读取从根节点到当前节点的对话历史**，不会受到其他分支内容的干扰。

#### 切换查看不同分支

- 中间面板展示了完整的对话树结构，每个节点都包含一条用户消息及其对应的 AI 回复。
- 单击任意节点，右侧面板会立即显示该节点的用户问题和 AI 回复内容。
- 你可以随时在不同分支之间跳转，继续深入讨论。

#### 管理多个对话树

- 左侧列表显示所有对话树，单击即可切换。
- 每个对话树都是完全独立存储的，互不影响。

#### 树形画布操作

- 如果树形结构过大，可以使用鼠标拖动画布的空白区域来移动视野。
- 使用滚动条或鼠标滚轮可以缩放画布（当前版本为 ScrollViewer 滚动，后续会升级为真正的缩放）。
- 节点会自动进行树形布局，新增节点后画布会自动刷新位置。

#### 常见问题

- **Q: 发送后没有收到回复？**  
  A: 请检查 API 配置是否正确（密钥、端点、模型名），并确保网络可以访问该 API 地址。
- **Q: 为什么我的分支追问 AI 似乎还是记得其他分支的内容？**  
  A: 请确认你单击选中了正确的父节点再发送新消息。如果直接在根节点下发送，会创建新的根节点分支，而不是基于当前分支。
